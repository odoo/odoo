# Cronograma Maestro del Proyecto

## Proyecto: Scrum y DevOps aplicado a un producto software open source

El presente cronograma maestro organiza el desarrollo del proyecto final del curso, desde la planificación inicial hasta la entrega final. El trabajo se estructura bajo la metodología Scrum, mediante un Sprint 0 de planificación y cuatro Sprints de desarrollo de 15 días calendario, integrando prácticas DevOps como control de versiones, automatización CI/CD, pruebas, despliegue a staging y publicación de documentación.
